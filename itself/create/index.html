milpa itself create

Create a milpa command

Usage

#

milpa itself create [options] NAME...

Arguments

#
  • NAME... required - the name you’d like for your command

Options

#
  • --description (string): a longer description of how this command works. Default: Works by doing a thing with another thing until the first thing looks about right.
  • --executable (bool): Create an empty, executable command. Useful when you’d like using something other than bash.
  • --open (bool): Open the script file in your current $EDITOR after creation.
  • --repo (string): a path to the milpa repo to create this command in. By default, the nearest .milpa directory from pwd and up.
  • --summary (string): a short summary of what this command does. Default: Does a thing.

Description

#

Creates a local milpa command named $NAME.

Global Options

#
  • --color (bool): Always print colors to stderr.
  • --help (bool): Display help for any command.
  • --no-color (bool): Disable printing of colors to stderr. Default: true.
  • --silent (bool): Silence non-error logging.
  • --skip-validation (bool): Do not validate any arguments or options.
  • --verbose (bool): Log verbose output to stderr.
